# Domain Yield Ranking

Recommended active path: empirical benchmark/data path, starting with benchmark leakage / ML evaluation fragility.

| Rank | Direction | Why it may yield stronger candidates | Required external sources | What would count as a real Fund | Main risks | First concrete experiment |
| ---: | --- | --- | --- | --- | --- | --- |
| 1 | Benchmark leakage / ML evaluation fragility | High measurement density; public datasets and protocols; deltas can be frozen before execution; independent holdouts and replays are practical. | OpenML tasks, scikit-learn examples, benchmark challenge notes, public split manifests, model cards, evaluation protocol docs. | A narrow checked refutation or externally review-ready discovery candidate showing a protocol/split/metric mechanism that survives baselines, rivals, counterexamples, holdout, and replay. | Metadata/provenance confounding; rediscovering known leakage; overclaiming benchmark-specific artifacts. | Run claim-first protocol perturbation on 8-12 public benchmark tasks with predeclared split/metric falsifiers and same-dataset negative controls. |
| 2 | Dataset provenance / data quality outcome effects | The system already has evidence-ref and holdout plumbing; public datasets give concrete target outcomes and repeatable controls. | UCI/OpenML/Kaggle-like public datasets where license/safety permits; public schema/data dictionaries; versioned dataset snapshots. | A bounded data-quality mechanism that predicts outcome degradation or residual behavior across independent datasets after simple baselines and provenance rivals are controlled. | Signals may be metadata-only or documentation/maturity dominated. | Pick 6 public datasets with known missingness/version/provenance metadata and freeze outcome-impact predictions before ablation. |
| 3 | Software reproduction mechanism discovery | Sovryn has repo/tooling infrastructure; failures can be mechanistic if maturity/docs/source-family controls are strict. | Public repos with test suites, issue labels, package metadata, CI configs, pinned dependencies. | A checked refutation/high-value reproduction mechanism where a nontrivial factor predicts replay outcomes beyond maturity/docs/source-family baselines. | Prior runs showed maturity/docs rivals are strong; risk of reproduction_fund_candidate classification. | Run matched-control reproduction mechanism tests on small public repos with predeclared failure mechanisms and maturity/docs controls. |
| 4 | Materials raw-data proxy experiments | Materials has domain value and public data, but previous Matbench package exposed raw-reproduction weaknesses. | Matbench/raw materials datasets, pymatgen/matminer descriptors, public train/test split manifests, descriptor configs. | A raw-data-reproducible descriptor/property residual with public featurizer, split, model, baseline, holdout, and counterexample manifests. | High risk of proxy-only signals and missing exact raw reproduction inputs. | Start with one raw Matbench task, public featurizer config, and frozen split before any candidate scoring. |
| 5 | Scientific public-data reliability | Real public data quality/reliability outcomes are measurable and inspectable; can support external review if claims stay narrow. | NASA/NOAA/ESA/open government scientific datasets, versioned public catalogs, public QA flags. | A bounded reliability/refutation candidate showing a repeated measured data-quality mechanism not explained by cadence/source-family baselines. | Often becomes ordinary data QA rather than discovery; external significance must be explicit. | Test a small cross-source public-data QA mechanism with source-family controls and independent temporal holdout. |
| 6 | Human-curated formal challenge claims | Formal path could work if a human supplies a high-value sharp claim, concrete object, and known-prior context. | Explicit claims from papers/issues/benchmark notes, concrete CNF/SMT/graph/automata/design/OEIS objects. | A checked proof/refutation or externally review-ready bounded conjecture with public source replay and known/triviality not fatal. | Requires human/domain expert input; autonomous source mining has not found suitable claims. | Accept one human-curated claim through the intake spec, then run the existing formal proof/witness path once. |
| 7 | Continue autonomous formal source-object mining | Low expected yield after repeated zero-yield loops; useful mainly as negative-control benchmark. | Same formal source pools already harvested: SAT/SMT, graph objects, automata, OEIS, designs. | A nonstandard replayable witness/refutation that scopes a rival and survives known/triviality. | Demonstrated diminishing returns; likely to keep generating no_valid_witness_or_counterexample. | Do not run unchanged; only use as regression/evaluation suite unless a curated external claim arrives. |
