# Updated Limitations

## Fund Semantics

`FUND_FOUND` is not a synonym for Nobel-ready discovery or Einstein-level
discovery. A Fund may be a reproduction, infrastructure, pipeline, or tool
capability result.

## SciPy Runtime Reproduction

The SciPy package remains a valid bounded reproduction Fund candidate. It does
not count toward Einstein/Nobel discovery score.

It proves:

- bounded runtime reproduction evidence can pass the existing Fund Gate
- the package can be inspected publicly
- the reproduction claim is bound to method, evidence, reproduce, and
  limitations artifacts

It does not prove:

- nontrivial new scientific insight across real targets
- domain scientific significance
- external validation or external adoption
- Nobel-level discovery
- Einstein-level discovery capability

## Scoring Limitation

Einstein/Nobel scoring now requires `discovery_fund_candidate` or
`externally_review_ready_discovery_candidate`. Reproduction, tool, pipeline, and
infrastructure Funds are excluded from that discovery score.
